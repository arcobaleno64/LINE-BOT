# LINE Bot 非工程師一頁操作卡

## 你要做的只有 4 步

1. 貼文件內容或上傳檔案
2. 指定模式（決策建議 或 刁鑽審稿）
3. 要求輸出責任人、期限、驗收方式
4. 看結論：可接受 / 有條件接受 / 退件

---

## 指令模板 A：決策建議

請用「決策建議與執行須知」模式回覆。
請固定輸出：
1. 決策結論（同意/有條件同意/不同意）
2. 決策理由（風險、成本、時程、治理責任）
3. 執行須知（責任人、期限、驗收方式）
4. 追問清單（需要補件的問題）

---

## 指令模板 B：刁鑽審稿

請用「文件審查（刁鑽嚴苛）」模式回覆。
先列重大缺失（高/中/低），再列必改清單，最後列退件標準。
每個問題都要附追問。
證據不足請直接寫：暫不接受，請依必改清單重提。

---

## 退件快速判定

任一成立就退件：
- 關鍵數據無來源
- 無責任人或無期限
- 無驗收方式
- 有資安風險但無控制措施

---

## 重審模板

請依上一輪必改清單重審。
先標示已修正與未修正項目。
最後只回答：可接受 / 有條件接受 / 退件。

---

## 最常見 3 個錯誤

- 只說「幫我看」：沒有指定模式，結果會太鬆
- 不給資料來源：容易被判證據不足
- 不要求期限與驗收：無法落地執行
